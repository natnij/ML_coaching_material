How to measure the quality of the prediction¶

The most commonly used performance metric for classification is accuracy = (number of true positive + number of true negative) / total number of samples

In [8]:
score = logisticRegr.score(x_test, y_test)
print('logistic regression has an accuracy of {:.2%}'.format(score))

logistic regression has an accuracy of 95.56%

But accuracy has serious problems when the sample classes are not balanced. https://stats.stackexchange.com/questions/312780/why-is-accuracy-not-the-best-measure-for-assessing-classification-models

Instead, we can use confusion matrix. It has the same number of rows and columns as the number of classes. It is very often used for visualization purposes. Each column of a confusion matrix represents a prediction class, the sum of that column is the total number of samples that are predicted to be in this class. Each row represents an actual class, the sum of that row is the total number of samples that are actually in this class. The numbers on the diagnal are the correct predictions (or the numerator of accuracy).

With unbalanced classes, precision and recall are better performance measurements.

Precision = true positive / (true positive + false positive), how many of the predicted positives are actually correct.

Recall = true positive / (true positive + false negative), how many of the actual positives are predicted correctly.

F-beta-score = ((1 + beta^2) Precision Recall) / (beta^2 * Precision + Recall), considers both precision and recall measurements, the higher the beta, the more importance is given to recall, or that a higher beta penalizes false negatives more than penalizing false positives.

ROC curve and AUC: for binary classes, by changing the decision threshold, we can plot the FP rate vs. TP rate and draw an Receiver Operating Characteristic (ROC) curve. AUC is the area below ROC, the bigger the AUC value, the better the classifier. https://developers.google.com/machine-learning/crash-course/classification/roc-and-auc

We can use the package from sklearn to calculate these performance measurements. Inputs are actual and predicted labels of the test set.

cm = metrics.confusion_matrix(y_test, predictions)
print(cm)
precision = metrics.precision_score(y_test, predictions, average='weighted')
recall = metrics.recall_score(y_test, predictions, average='weighted')
fbeta = metrics.fbeta_score(y_test, predictions, average='weighted',beta=1)
print('Precision:{:0.2f}, recall:{:0.2f}, fbeta:{:0.2f}'
      .format(precision,recall,fbeta))

y_test_binary = [0 if x < 5 else 1 for x in y_test]

auc = metrics.auc(fpr, tpr)
plt.plot(fpr,tpr,label="auc="+str(auc))

The importance of data preprocessing¶

Method1: scaling

Scaling adjusts input data range and variance, such that all features have a mean of 0 and a variance of 1. This is for models that specifically require such inputs.

In [18]:
from sklearn.preprocessing import StandardScaler
scaler = StandardScaler()
scaler.fit(x_train)
x_train_scaled = scaler.transform(x_train)
x_test_scaled = scaler.transform(x_test)

Method2: principle component decomposition (PCA)

PCA is used for rendering features linearly independent, and for reducing feature dimension. Note that PCA does not reduce nonlinear dependency.

In [19]:
from sklearn.decomposition import PCA
pca = PCA(.99)
pca.fit(x_train_scaled)
x_train_decomposed = pca.transform(x_train_scaled)
x_test_decomposed = pca.transform(x_test_scaled)

Now if we use the Naive Bayes model for prediction again, the accuracy is improved:
