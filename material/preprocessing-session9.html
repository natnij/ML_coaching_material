def calDistance(dataRow, centers):

Dealing with imbalanced classes¶

When one class has much more samples than other classes. This will result in high accuracy in major class and low accuracy in minor classes.

Some classifiers are worse in dealing with imbalanced data than others, for example logistic regression. Decision trees and ensemble methods are much better.

Here is an explanation of class distribution requirements in logistic regression: https://medium.com/bluekiri/dealing-with-highly-imbalanced-classes-7e36330250bc

The post also mentions solutions to data imbalance, e.g.,

  • undersampling of the major class with the disadvantage of less data efficiency,
  • oversampling of the minor classes with the risk of overfitting,
  • adding sample weight or class weight in the algorithm, such as mentioned here: https://stackoverflow.com/questions/32492550/what-is-the-difference-between-sample-weight-and-class-weight-options-in-scikit

Scaling¶

When features have different unit of measurements, the feature space is big for gradient descent methods to find the optimal (i.e., minimization or maximization), or it may never converge. Therefore we scale the data.

Different scalers are offered by sklearn: https://scikit-learn.org/stable/modules/preprocessing.html

Method1: min-max: $x_{norm} = \cfrac{x-x_{min}}{x_{max}-x_{min}}$

Method2: Standard: turn data into mean=0, variance=1: $x_{norm} = \cfrac{x-\mu}{\sigma}$. The scaler works best when the original data distribution is close to normal distribution.

Method2: Box-cox: https://www.statisticshowto.datasciencecentral.com/box-cox-transformation

$ y(\lambda) = \begin{cases} \cfrac{y^{\lambda}-1}{\lambda} &\quad \text{if } \lambda \neq 0 \\ \text{log }y &\quad \text{if } \lambda = 0 \end{cases}$

There are three boxcox functions in scipy.stats:

  • boxcox_normmaxcalculates the best $\lambda$
  • boxcox_normplot is a grid search between upper and lower bounds of $\lambda$. We choose the one with the biggest ppcc value.
  • boxcox returns transformed data given $\lambda$, or guess the best $\lambda$ if it is not given and return the transformed data.

Feature selection¶

In principal, choose features with a high variance, and features that are correlated to the dependent variable (the target).

Why select features? Because too many features may lead to overfitting: https://towardsdatascience.com/understanding-the-bias-variance-tradeoff-165e6942b229

Simply put: suppose $y=f(x)+\epsilon$$\epsilon$ is a normally distributed random error with mean=0, the model is $\hat{f}(x)$.

Total error $\text{Err}(x)=\mathbf{E}[(y-\hat{f}(x))^2]$ can be broken down into:

$\text{Err}(x)=\big( \mathbf{E}[\hat{f}(x)]-f(x) \big)^2 + \mathbf{E}\big[ (\hat{f}(x)-\mathbf{E}[\hat{f}(x)])^2 \big] + \sigma_{e}^2$,

or $\text{Err}(x)=\text{Bias}^2+\text{Variance}+\text{Irreducible Error}$

Bias measures how much the prediction is "off" from the actual. Variance measures how dispersed the prediction is.

As complexity of $\hat{f}(x)$ increases, bias decreases (model fits better to the actual and prediction is closer to the actual), variance increases (the better the model fits existing data, the more sensitive it is to new data that is maybe a bit different). To minimize $\text{Err}(x)$, we need to find the optimum number of features.

Method1: filters: ignore features with low variance. It is computationally cheap. It disregards correlation between feature and target variables and therefore should not be used without checking variable correlation.

Method2: univariate selection: selection based on univariate statistical tests.

ANOVA f-test: applies to continuous feature and categorical target (discrete target means classification problems), as pointed out on sklearn: https://scikit-learn.org/stable/modules/generated/sklearn.feature_selection.SelectKBest.html. A very intuitive and gentle understanding of how f_classif selects features is here: https://datascience.stackexchange.com/questions/74465/how-to-understand-anova-f-for-feature-selection-in-python-sklearn-selectkbest-w

One-way ANOVA is originally a statistic tool that is used for determining the response of a continuous target to a discrete independent variable (yes I think the sklearn f_classif function is using the concept the other way around). Note that in ANOVA the discrete variable does not have to be categorical, it can also be nominal (meaning it's discrete numbers but you can order them). To use ANOVA, there are some assumptions to the data distribution: the continuous data corresponding to the different classes should all be normally distributed with similar variances. Otherwise it impacts the performance of the ANOVA test.

If you would like to understand the math, here are some details on ANOVA (not on f_classif): https://towardsdatascience.com/anova-for-feature-selection-in-machine-learning-d9305e228476

Also note that ANOVA is usually used when you have at least three classes in the discrete variable. If you have less than 3, consider using t-test: https://www.analyticsvidhya.com/blog/2021/07/t-test-performing-hypothesis-testing-with-python/

FYI: n-way ANOVA is used for determining the response of a continuous target to multiple discrete independent variables. We don't use this analysis too often, so we'll just ignore it for the time being.

A side note: understanding PCA from SVD:

If the feature matrix is $M \in \mathbb{R}^{m*n}$, via SVD it is decomposed into $M = U\Sigma V^T, U \in \mathbb{R}^{m*m}, \Sigma \in \mathbb{R}^{m*n}, V \in \mathbb{R}^{n*n}$.

$\Sigma$ is a diagonal matrix corresponding to each feature's singular values, the higher the value, the more important the feature.

If we sort $U,V$'s columns and $\Sigma$'s diagonal values by descending order of the singular values, we choose the first $k$ features by keeping the first $k$ columns of matrices $U,V$, and keeping the first $k$ diagonal values of $\Sigma$.

Then we have: $M \in \mathbb{R}^{m*k}, \Sigma \in \mathbb{R}^{k*k}, M \in \mathbb{R}^{n*k}$.

For any new data point with n features $x \in \mathbb{R}^{1*n}$, we can transform $x$ in to the new vector with k features by: $xV \in \mathbb{R}^{1*k}$.

In both victim race categories, black inmates' probability of receiving capital punishment is much higher than white inmates.

In plain English: white inmates' murder victims are mostly also white, and statistically speaking, in cases where the victim is a white, it is overall likely that the inmate receives capital punishment. So when the victims' race data is aggregated, we see white murderers getting more capital punishments -- because they happen to have murdered more white victims.

In fact, a black inmate murdering a white victim is more than 2 times as likely to receive capital punishment comared to a white inmate, and at least until 2014, no white inmate in Florida has received capital punishment for murdering a black victim.

This is what is called mixed effects / multilevel models. The name multilevel comes from when samples are nested within batch, e.g., in HR data you have repeating hiarchical feature such as "department", with another feature that is actually a subcategory of "department". In this case, the two features are strongly correlated, and that violates a lot of regression models' data assumptions.

The name mixed effects come from a more generalized definition. A fixed effects model refers to a regression model in which the group means are fixed (non-random) as opposed to a random effects model in which the group means are a random sample from a population. For example, the mean of all employees' performance in a department is fixed, however in different departments, the means are "random effects" drawn from a distribution. But random effects do not have to be related to only the hierarchy. Therefore both terms are usually used interchangeablly. Reference see here: https://stats.stackexchange.com/questions/171313/difference-between-multilevel-modelling-and-mixed-effects-models

Sometimes it is also called a "drifting covariate", meaning there is a potentially unknown covariate (feature), that is causing the drift in the mean of some other features. Failure to address the covariate may result in completely wrong conclusions, as we see in the example above.

More formal ways to deal with mixed effects are multi-level modeling MLM and generalized estimating equations GEE, some discussions here: https://www.quora.com/How-can-we-include-hierarchical-data-such-as-category-and-subcategory-as-independent-variables-in-a-statistical-or-machine-learning-model

As explained in this post: https://www.propublica.org/article/how-we-analyzed-the-compas-recidivism-algorithm, and a project in the US tracks police use of facial recognision tools and analyzes for bias: https://www.perpetuallineup.org/findings/racial-bias

  2. Due to imbalanced classes in sample data and inappropriate choice of performance metrics, models are applied in real life that has severe racial impacts. For example, if a facial recognition software for predicting recidivism has minimization of false negative or false negative values, it has different impacts on racial groups, see details here: https://www.districtdatalabs.com/fairness-and-bias-in-algorithms
